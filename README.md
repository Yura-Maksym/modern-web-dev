# Лабораторна робота №1: Шаблони проєктування
**Курс:** Modern Web Development Technologies

Це виконання першої лабораторної роботи, де я реалізував три патерни на мові Python. Усі приклади демонструють розділення логіки та гнучкість архітектури.

## 1. Factory Method (Фабричний метод) — Породжувальний
* **Проблема:** Клас не знає заздалегідь, об'єкти яких підкласів йому потрібно створювати. 
* **Ідея реалізації:** Ми створюємо абстрактний клас `Logistics` з методом `create_transport()`. Конкретні класи (`RoadLogistics`, `SeaLogistics`) перевизначають цей метод і повертають відповідний об'єкт транспорту (`Truck` або `Ship`). Таким чином, основна програма працює з інтерфейсом `Transport`, а не з конкретними машинами.
* **UML-діаграма:**

![Factory Method Diagram](https://refactoring.guru/images/patterns/diagrams/factory-method/example.png)

* **Застосування:** Системи логістики, кросплатформні UI-компоненти.

## 2. Facade (Фасад) — Структурний
* [cite_start]**Проблема:** Система складається з багатьох складних підсистем (Audio, Video), і взаємодія з ними напряму робить код заплутаним. 
* **Ідея реалізації:** Клас `VideoEditorFacade` слугує єдиною точкою входу. Він ініціалізує `AudioMixer` та `VideoConverter` всередині себе. Клієнту достатньо викликати один метод `render()`, а Фасад сам керує складними викликами підсистем.
* **UML-діаграма:**

![Facade Diagram](https://refactoring.guru/images/patterns/diagrams/facade/example.png)

* **Застосування:** Складні API, бібліотеки для роботи з графікою чи медіа.

## 3. Observer (Спостерігач) — Поведінковий
* **Проблема:** Зміна стану одного об'єкта потребує оновлення невідомої кількості інших об'єктів. 
* **Ідея реалізації:** Клас `Publisher` зберігає список об'єктів-підписників. Метод `subscribe()` додає нових слухачів, а `notify()` проходить циклом по списку і викликає метод `update()` у кожного підписника, коли з'являється нова новина.
* **UML-діаграма:**

![Observer Diagram](https://refactoring.guru/images/patterns/diagrams/observer/example.png)

* **Застосування:** Push-сповіщення, підписки на новини, події в інтерфейсі.

## Демонстрація роботи
Після запуску `main.py` програма імітує:
1. Створення транспорту через фабрику.
2. Процес рендеру відео через один виклик фасаду.
3. Розсилку повідомлення двом підписникам одночасно.
